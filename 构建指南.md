## 三、SSH 远程控制（笔记本）

### 3.1 安装 OpenSSH Server（笔记本管理员 PowerShell，仅一次）

```powershell
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
Set-Service sshd -StartupType Automatic
Start-Service sshd
New-NetFirewallRule -Name OpenSSH -DisplayName OpenSSH -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22
```

### 3.2 配置免密登录（从台式机到笔记本）

**踩坑要点**：
- 笔记本用户 `xiaohai` 的 profile 目录实际是 `C:\Users\Administrator`，不是 `C:\Users\XIAOHAI`
- `authorized_keys` 权限必须只有 `SYSTEM` 和用户本人，不能有 `Administrators` 组
- 私钥不能有 passphrase

**步骤**：

1. 台式机生成密钥（无密码）：
```powershell
cmd /c 'ssh-keygen -t ed25519 -f %USERPROFILE%\.ssh\id_ed25519 -N "" -q'
```

2. 把公钥写入笔记本。在笔记本上（管理员 PowerShell）：
```powershell
New-Item -ItemType Directory -Path "C:\Users\Administrator\.ssh" -Force
notepad "C:\Users\Administrator\.ssh\authorized_keys"
```
粘贴台式机 `id_ed25519.pub` 内容（一整行，无换行），保存。

3. 修复权限：
```powershell
icacls "C:\Users\Administrator\.ssh\authorized_keys" /inheritance:r /grant SYSTEM:(F) /grant xiaohai:(F)
icacls "C:\Users\Administrator\.ssh" /inheritance:r /grant SYSTEM:(F) /grant xiaohai:(F)
```

4. 台式机测试：
```powershell
ssh XIAOHAI@192.168.10.121 hostname
# 应返回 xiaohaifale
```

### 3.3 WMI 备用方案

当 SSH 不可用时（无需安装任何东西）：

```powershell
# 远程执行命令
wmic /node:192.168.10.121 /user:XIAOHAI /password:<密码> process call create "你要跑的命令"

# 查询信息
wmic /node:192.168.10.121 /user:XIAOHAI /password:<密码> computersystem get name
```

**WMI 限制**：看不到 stdout，复杂操作用 base64 编码 PowerShell 通过 `-EncodedCommand` 传入。

## 四、AI 视觉操控（helix-pilot + Ollama）

### 4.1 架构

```
台式机 Reasonix ──MCP/HTTP──→ 笔记本 helix-pilot ──→ 笔记本 Ollama (moondream)
                                        │
                                        └──→ pyautogui (截图/点击/打字)
```

### 4.2 笔记本安装 Ollama

```powershell
winget install Ollama.Ollama --accept-package-agreements --silent
```

首次运行会触发升级（约 1 分钟），等待完成。

### 4.3 下载视觉模型

Ollama 服务启动后：
```cmd
set PATH=%PATH%;C:\Users\Administrator\AppData\Local\Programs\Ollama
ollama pull moondream:latest
```

模型约 1.7GB，仅 CPU 推理（Intel Iris Xe 核显被 Ollama 默认跳过）。

如网络不通，可通过 HTTP_PROXY 环境变量走代理下载。

### 4.4 安装 helix-pilot

```powershell
git clone https://github.com/tsunamayo7/helix-pilot.git C:\Users\Administrator\helix-pilot
cd C:\Users\Administrator\helix-pilot
uv sync
```

**注意**：如果 `git clone` 不通，可以从台式机 scp：
```powershell
scp -r XIAOHAI@192.168.10.121:"C:/Users/Administrator/helix-pilot" .
```

### 4.5 配置

编辑 `C:\Users\Administrator\helix-pilot\config\helix_pilot.json`：
```json
{
  "ollama_endpoint": "http://127.0.0.1:11434",
  "vision_model": "moondream:latest",
  "safe_mode": true,
  "execution_mode": "apply_with_approval"
}
```

### 4.6 启动 HTTP 模式

helix-pilot 默认是 stdio MCP，需要改为 HTTP 模式才能被远程连接。

编辑 `server.py` 最后一行，把 `mcp.run()` 改为：
```python
mcp.run(transport="http", host="0.0.0.0", port=8765)
```

保存为 `server_http.py`，启动：
```cmd
cd C:\Users\Administrator\helix-pilot
python server_http.py
```

开放防火墙：
```cmd
netsh advfirewall firewall add rule name="helix_pilot" dir=in protocol=TCP localport=8765 action=allow
```

### 4.7 台式机连接

Reasonix 注册 MCP 服务器：
```
名称:   laptop-pilot
传输:   http
URL:    http://192.168.10.121:8765/mcp
```

提供 20 个工具：`screenshot` `click` `type_text` `hotkey` `scroll` `describe` `find` `verify` `auto` `browse` `status` `list_windows` `wait_stable` 等。

**注意**：笔记本重启 server_http.py 后，台式机需要重新注册 MCP（session 会失效）。

## 五、自动启停（笔记本）

笔记本的 helix-pilot 通过 Windows 计划任务 + WLAN 事件触发器自动管理。

### 5.1 自动启停脚本

文件：`C:\Users\Administrator\helix-pilot\auto_pilot.ps1`

```powershell
$targetSSID = "fale520"
$ollamaExe = "C:\Users\Administrator\AppData\Local\Programs\Ollama\ollama.exe"
$pilotDir = "C:\Users\Administrator\helix-pilot"

$wifi = netsh wlan show interfaces | Select-String "SSID" | Select-String -NotMatch "BSSID"
$currentSSID = ($wifi -replace ".*:\s*", "").Trim()

if ($currentSSID -eq $targetSSID) {
    # 连上目标 WiFi → 启动
    if (-not (Get-Process -Name "ollama" -ErrorAction SilentlyContinue)) {
        Start-Process $ollamaExe -ArgumentList "serve" -WindowStyle Hidden
        Start-Sleep 3
    }
    $already = Get-WmiObject Win32_Process -Filter "Name like 'python%'" |
        Where-Object { $_.CommandLine -like "*server_http*" }
    if (-not $already) {
        Start-Process python -ArgumentList "server_http.py" -WorkingDirectory $pilotDir -WindowStyle Hidden
    }
} else {
    # 断开 WiFi → 关闭
    Get-WmiObject Win32_Process -Filter "Name like 'python%'" |
        Where-Object { $_.CommandLine -like "*server_http*" } |
        ForEach-Object { Stop-Process -Id $_.ProcessId -Force }
    Get-Process -Name "ollama" -ErrorAction SilentlyContinue |
        Stop-Process -Force
}
```

### 5.2 计划任务

任务名：`AutoPilot_fale520`

| 触发器 | 说明 |
|--------|------|
| WLAN EventID 8001 | WiFi 连接成功 |
| WLAN EventID 8003 | WiFi 断开 |
| 开机 | 后备 |

使用 PowerShell 创建：
```powershell
$action = New-ScheduledTaskAction -Execute "powershell.exe" `
    -Argument "-ExecutionPolicy Bypass -WindowStyle Hidden -File C:\Users\Administrator\helix-pilot\auto_pilot.ps1"

# WLAN 事件触发器
$CIMTriggerClass = Get-CimClass -Namespace Root/Microsoft/Windows/TaskScheduler -ClassName MSFT_TaskEventTrigger

$connectTrigger = New-CimInstance -CimClass $CIMTriggerClass -ClientOnly
$connectTrigger.Enabled = $true
$connectTrigger.Subscription = @"
<QueryList><Query Id='0' Path='Microsoft-Windows-WLAN-AutoConfig/Operational'>
<Select Path='Microsoft-Windows-WLAN-AutoConfig/Operational'>
*[System[Provider[@Name='Microsoft-Windows-WLAN-AutoConfig'] and EventID=8001]]
</Select></Query></QueryList>
"@

# 同上创建 EventID=8003 的断开触发器

$bootTrigger = New-ScheduledTaskTrigger -AtStartup
$principal = New-ScheduledTaskPrincipal -UserId "SYSTEM" -RunLevel Highest

Register-ScheduledTask -TaskName "AutoPilot_fale520" `
    -Action $action -Trigger $connectTrigger,$disconnectTrigger,$bootTrigger `
    -Principal $principal -Force
```

1. 两台连上 fale520 WiFi

## 八、踩坑总汇

| 问题 | 原因 | 解决 |
|------|------|------|
| Ollama 模型下载超时 | 代理未覆盖 ollama.com | 配好代理或设 HTTP_PROXY 环境变量 |
| pip/uv 下载超时 | PyPI 直连不通 | 用清华镜像加速 |
| SSH 公钥被拒 | authorized_keys 在错误目录或有错误权限 | 放 `C:\Users\Administrator\.ssh\`，权限只留 SYSTEM+用户 |
| SSH 免密失败 | 私钥有 passphrase | `ssh-keygen -N ""` 生成无密码密钥 |
| Ollama 返回 407 | Windows 系统代理拦截 localhost | 关闭系统代理或设 NO_PROXY |
| WMI/SSH 无法启 GUI | 服务会话无桌面 | 让用户在桌面 CMD 里手动跑 |
| MCP session 丢失 | 笔记本重启 server 后 session 失效 | 台式机重新注册 MCP |
| Deskflow 连不上 | 防火墙或 AP 隔离 | 加防火墙规则、检查路由器 |
| Ollama serve 启动失败 | 非交互会话无 WebView2 | 从桌面会话启动 |
