# 笔记本远程操控方案

从台式机通过 MCP 协议远程操控笔记本 GUI——截图、看图识别、点击、打字、自动化任务。

---

## 架构

```
台式机 (Reasonix)                    笔记本 (xiaohaifale)
     │                                      │
     │  MCP over HTTP (:8765)               │
     ├─────────────────────────────────────→│
     │                                      ├── helix-pilot (FastMCP HTTP server)
     │                                      ├── Ollama (moondream 视觉模型)
     │                                      └── pyautogui (键鼠控制)
     │                                      
     路由器 (192.168.10.1)
     └── fale520 WiFi
```

### 前置依赖

- Python 3.12 + Git + uv（已装）
- Ollama（已装，`C:\Users\Administrator\AppData\Local\Programs\Ollama\`）

### helix-pilot

```
路径: C:\Users\Administrator\helix-pilot\
配置: config\helix_pilot.json  → ollama_endpoint: http://127.0.0.1:11434
模型: moondream:latest (1.7GB, CPU推理, 视觉问答)
```

```cmd
cd C:\Users\Administrator\helix-pilot
python server_http.py
```

启动后监听 `0.0.0.0:8765`，防火墙已开放。

## 自动启停

笔记本创建了计划任务 `AutoPilot_fale520`：

| 触发器 | 事件 |
|--------|------|
| 连上 fale520 | WLAN EventID 8001 |
| 断开 fale520 | WLAN EventID 8003 |
| 开机 | 后备 |

脚本：`C:\Users\Administrator\helix-pilot\auto_pilot.ps1`

逻辑：
- 检测到 fale520 → 启动 Ollama serve + helix-pilot HTTP server
- 离开 fale520 → 关闭两个进程释放内存

## 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| Ollama 模型下载超时 | 目标域名需要走代理 | 设 HTTP_PROXY 环境变量 |
| pip 安装超时 | PyPI 直连慢 | 用 `uv sync` 走镜像源 |
| Ollama 连不上 (407) | 系统代理拦截 localhost | 关掉系统代理 `ProxyEnable=0` |
| WMI/SSH 无法启动 GUI | 服务会话无桌面 | 用户桌面 CMD 手动跑 |
| SSH 公钥被拒 | Profile 目录不对 + 权限过宽 | 放正确目录，权限只留 SYSTEM+用户 |
| 私钥有密码 | BatchMode 无法交互 | 生成无密码密钥 |
| MCP session 丢失 | 服务重启后 session 失效 | 台式机重新注册 MCP |
